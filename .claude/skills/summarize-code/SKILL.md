name: summarize-code
description: Summarizes how code in a repository works for a TSE.

---

Steps to follow:

1. **Ask for path to the code repository**: Access the code 
2. **Walk through the code**: Understand code structure, how the code works, what the code does, and required inputs and outputs
3. **Explain the code**: Start with a 1-3 sentence high-level overview of what the code does. Then, expand on the specifics assuming your target audience is a technical solutions engineer who is unfamiliar with the code but needs to learn what the code does and how to use it.
4. **Explain the steps**: Explain the steps for how to use the code
5. **Walkthrough an example**: Walkthrough an example of how to use the code (or an existing example if that exists) and explain what is happening at each step from the code perspective.

Keep explanations conversational. For complex concepts, use examples.
